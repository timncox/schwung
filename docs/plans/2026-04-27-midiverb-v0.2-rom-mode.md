# Midiverb v0.2 — ROM Mode + Expanded Controls

> **For Claude:** Execute task-by-task with `superpowers:executing-plans` discipline.

**Goal:** Add ROM-loading path (cycle-accurate cycle interpreter via upstream's `dasp16.h`), proper LFO support for LFO-using programs, and 4 new real-time controls (Damping, LFO Rate/Depth, Saturation, Tilt).

**Architecture:** Auto-detect — if ROM file exists at `modules/audio_fx/midiverb/roms/<unit>.rom`, use cycle-accurate path; else fall back to decompiled (current v0.1 behavior). Both paths share my existing resampler / scaling / mix / filter chain, so DSP quality regression is impossible.

**Tech stack:** vendor `dasp16.h` + `lfo.h` + `rom.h` from upstream `thement/midiverb_emulator`. Use schwung-manager's `assets` config (mirroring schwung-jv880) for ROM uploads.

## Phase 2 — ROM loader path (Tasks 3-5)

### Task 3: Add ROM-mode state to `mv_instance_t`

Add to `midiverb_core.h`:
```c
typedef enum { MV_SOURCE_DECOMPILED = 0, MV_SOURCE_ROM = 1 } mv_source_t;

/* Inside mv_instance_t: */
mv_source_t  source;          /* which DSP path to run */
Machine      machine;         /* cycle-accurate state when source=ROM */
const char  *module_dir;      /* set in create_instance, used to resolve rom paths */
char         rom_status[64];  /* e.g. "ROM (Midiverb)" or "Decompiled (no ROM)" */
```

Initialize: `source = MV_SOURCE_DECOMPILED`, `module_dir` snapshot in `mv_create`.

### Task 4: Implement `mv_try_load_rom(inst)` in `midiverb_core.c`

Behavior:
1. Build path: `<module_dir>/roms/<unit_filename>` where unit_filename is `midiverb.rom` / `midifex.rom` / `midiverb2.rom`
2. If `access(path, R_OK) != 0`: return 0 (no ROM)
3. Otherwise: pick the right `RomType` from upstream's `rom_types[]`, call `load_rom(&inst->machine, ...)` for the current program, set `source = MV_SOURCE_ROM`
4. Update `rom_status` string

Called from `apply_pending` after a unit/program change.

### Task 5: Branch in process_block

In the per-sample inner loop:
```c
int16_t ol = 0, or_ = 0;
if (inst->source == MV_SOURCE_ROM) {
    Sample out;
    run_machine_tick(&inst->machine, input_mono, &out);
    ol = out.s[0]; or_ = out.s[1];
} else {
    inst->effect_fn(input_mono, &ol, &or_, inst->dram, inst->ptr,
                    inst->lfo1, inst->lfo2);
}
```

Both paths produce identical 13-bit int16 stereo. Everything downstream (8x output scaling, LPF, width, upsample, mix) stays identical.

Add `source` and `rom_status` keys to `mv_get_param` for UI display.

## Phase 3 — LFO support (Tasks 6-7)

### Task 6: Wire upstream's `init_lfo_for_program` + `patch_machine`

For ROM mode (cycle-accurate), follow upstream's main loop:
- At program change (in `apply_pending`): call `init_lfo_for_program(program_number, &lfo1, &lfo2, &lfo_patch)`. Store `run_lfo` flag.
- Per sample, when `(sample_counter & 7) == 0` and `run_lfo`: call `lfo1.update()` / `lfo2.update()`, OR with `lfo_patch->top1/top2`, call `patch_machine(...)`.

For decompiled mode, the LFO arguments to `effect_fn` get the same lfo1/lfo2 update treatment (no `patch_machine` since program is fixed — but the LFO values still affect any flanger that uses them).

### Task 7: LFO Rate + LFO Depth params

- `lfo_rate` — float 0.0..4.0, default 1.0 (multiplier on natural rate). Apply to `lfo1.update_rate / lfo2.update_rate` if upstream's LFO struct supports it; otherwise modulate the per-update increment.
- `lfo_depth` — float 0.0..1.0, default 1.0 (scales the LFO value sent to effect / patch).

Both no-op for non-LFO programs (gracefully).

## Phase 4 — New controls (Tasks 8-10)

### Task 8: Damping (LPF in feedback loop)

Add `damping` param 0.0..1.0, default 0.0 (no damping). Apply a one-pole LPF inside the feedback path:

```c
/* Before feeding fb into next sample's mono: */
inst->fb_lpf_l = a_damp * inst->fb_lpf_l + (1.0f - a_damp) * inst->fb_l;
inst->fb_lpf_r = a_damp * inst->fb_lpf_r + (1.0f - a_damp) * inst->fb_r;
/* Then use inst->fb_lpf_l/r in the mono+fb sum instead of inst->fb_l/r */
```

Where `a_damp` = `damping * 0.99 + 0.01` (clamped to avoid divide-by-zero), gives cutoff sweeping from ~3 kHz (damping=0) to ~50 Hz (damping=1) at 23.4 kHz.

### Task 9: Saturation (soft-clip on wet)

Add `saturation` param 0.0..1.0, default 0.0. Apply tanh-style soft clip post-effect, pre-LPF:

```c
if (inst->saturation > 0.0f) {
    float drive = 1.0f + 4.0f * inst->saturation;
    wl = tanhf(wl * drive) / tanhf(drive);
    wr = tanhf(wr * drive) / tanhf(drive);
}
```

Adds gentle harmonics at low values, real grit at high values. Cheap.

### Task 10: Tilt (single knob replacing root low_cut/high_cut)

Add `tilt` param -1.0..1.0, default 0.0 (neutral). Internally drives both filters:

```c
/* tilt < 0 → cut highs (darken). tilt > 0 → cut lows (brighten). */
float low_hz  = inst->low_cut_hz;
float high_hz = inst->high_cut_hz;
if (inst->tilt > 0.0f) {
    /* Brighter: raise low cut from base toward 1 kHz */
    low_hz = inst->low_cut_hz + inst->tilt * (1000.0f - inst->low_cut_hz);
} else if (inst->tilt < 0.0f) {
    /* Darker: lower high cut from base toward 1 kHz */
    high_hz = inst->high_cut_hz + inst->tilt * (inst->high_cut_hz - 1000.0f);
}
/* Use low_hz / high_hz as the actual filter cutoffs */
```

Keep `low_cut_hz` and `high_cut_hz` as separate params (in Levels submenu) for power users.
